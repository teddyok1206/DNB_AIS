## 범위
- `[A]` 코드에서 GeoTIFF 밝기 압축식을 확인
- DRUID는 기존 compressed GeoTIFF를 그대로 사용하고, GAT brightness feature만 inverse 값으로 바꾸도록 수정
- inverse 적용이 실제 graph feature에만 들어가는지 검증

## 참고한 `[A]` 압축식
- [A]_dnb2geotif_v2_modified_4326_metadata.py#L268](/Users/jungtaeuk/Desktop/SATGEO/[DNB_AIS]/[3]_DNB_AIS%20-%20(STEP%203)/[A]_dnb2geotif_v2_modified_4326_metadata.py#L268)
- 핵심 식은 다음과 같다.
  - `grid_brightness = (2/np.pi) * np.arctan(grid_brightness / 1e-9)`
- 따라서 inverse는
  - `radiance = 1e-9 * tan((pi/2) * brightness)`

## 동작 방식
- DRUID 입력: 기존 `scene.image` 그대로 사용
- DRUID contour membership: 기존과 동일
- GAT brightness feature: compressed 값 대신 inverse radiance 사용
- 좌표 feature `x, y`와 target `y`는 기존과 동일
- reverse 후에는 사용자 요청대로 추가 normalize를 하지 않는다.

## 검증 결과
- 간단 검증에서 첫 cluster의 brightness 범위는 다음과 같았다.
  - compressed: `3.155443e-01 ~ 9.581851e-01`
  - inverse radiance: `5.406751e-10 ~ 1.520283e-08`
  - graph feature 실제 입력: `5.406751e-10 ~ 1.520283e-08`
- scene 전체 DRUID 입력 범위는 여전히 compressed TIFF 값이었다.
  - `1.666920e-01 ~ 9.979022e-01`

## 현재 판단
- 요청한 대로 DRUID는 건드리지 않고, GAT brightness만 원래 dynamic range로 복원했다.
- 다음부터 나오는 모델 결과 해석은 기존 compressed brightness가 아니라 inverse radiance feature를 기준으로 봐야 한다.
- 추가 normalize가 없으므로, 이후 학습 안정성이나 weighting이 다시 달라질 수 있다.
