# IEEE Draft Writing Template

Use this as a lightweight drafting scaffold before transferring text into the IEEE `.docx` template.

## Section Checklist

For each section, write in this order:

1. Claim: one sentence stating what this section proves or explains.
2. Evidence: data, method detail, figure, table, or citation supporting the claim.
3. Boundary: what is not claimed.
4. Link: transition to the next section.

## Result Placeholder Convention

Use explicit placeholders until final experiment results are available:

```text
TODO_RESULT_<metric_or_table_name>
TODO_FIGURE_<figure_name>
TODO_TABLE_<table_name>
```

Do not invent final numeric values. Pull final values from committed evaluation JSON summaries only.

## Figure Decision Labels

- `planned`: candidate figure to review later.
- `included`: accepted figure intended for the current report draft.
- `removed`: figure tried and rejected, kept only for traceability.
