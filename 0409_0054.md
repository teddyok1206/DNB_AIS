## directed edge 검토

결론부터 말하면, 현재 규모에서는 GT spreading을 위해 굳이 directed edge로 바꿀 필요는 크지 않다.

현재 `batch_demo` 기준 수치:

- graph 수: `125`
- 총 node 수: `6375`
- 총 edge 수: `145748`
- 평균 edge 수/graph: `1165.984`
- median edge 수/graph: `538`
- 최대 edge 수/graph: `13182`
- 평균 node 수/graph: `51`
- 최대 node 수/graph: `449`
- 평균 degree: `22.86`
- raw positive source 수: `79`

현재 spreading은 graph마다 raw positive source에서 BFS를 돌려 `1.0 -> 0.6 -> 0.2`로 퍼뜨린다. 대략적인 비용 감각으로 보면 source 수 `79`개에 대해 edge 집합 `145,748`를 훑는 수준이라, 총 edge 방문량은 거칠게 `79 x 145,748 ≈ 1.15e7` 수준이다. 이 정도는 지금 파이프라인 전체 비용에서 지배적인 수준이 아니다.

그래서 현재 판단은 다음과 같다.

- `속도만` 이유로 directed edge로 바꿀 필요는 없음
- undirected로 두는 편이 구현과 해석이 단순함
- 특히 지금 spreading 목적이 “영향 범위 부여”이므로 방향성을 굳이 강제할 이유도 약함

다만 아래 조건에서는 재검토 가치가 있다.

- `KR`처럼 scene이 더 커질 때
- `radius_pixels`를 더 키울 때
- hop 수를 늘릴 때
- cluster 수가 지금보다 크게 늘 때

현재 단계에선 undirected 유지가 맞다.
