## MPS 원인 확인

- 방금 MPS 테스트를 sandbox 밖에서 다시 실행했다.
- 결과:
  - `torch 2.9.1`
  - `mps_built = True`
  - `mps_available = True`
  - 실제 `torch.device('mps')` tensor 생성과 계산도 성공

실행 결과:

- python 경로: `/Users/jungtaeuk/anaconda3/envs/DNB_AIS/bin/python`
- macOS: `15.7.3`, `arm64`
- test output:
  - `device mps:0`
  - `y [1.0, 4.0, 9.0, 16.0, 25.0]`

해석:

- 사용자 머신이나 `DNB_AIS` 환경 자체가 깨진 것이 아니다.
- Codex tool의 기본 sandbox 안에서는 MPS가 비활성으로 보였고,
- unsandboxed 실행에서는 정상 동작했다.

결론:

- 어제 동작했고 지금도 로컬 환경에선 되는 것이 맞다.
- 오늘 Codex가 기본 sandbox로 실행한 테스트만 거짓 음성(false negative)이었다.
- 이후 GPU/MPS가 필요한 학습·추론은 sandbox 밖 실행으로 진행해야 한다.
