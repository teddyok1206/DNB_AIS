# 0409_1418

## 방향성 정리

- `edge weighting`은 두 층으로 나뉜다.
  - graph 위에서 GT를 퍼뜨리는 `edge-decay GT`
  - 학습 loss에서 양성/중심부를 더 세게 보는 `positive_weight`, `count_weight_alpha`
- 현재 메인 기본값에서 제거한 것은 후자다.
- 유지되는 것은 전자다. 즉 `GT 값이 graph edge를 따라 퍼져 나가는 구조`는 그대로 유지된다.

## 사용자 우선순위 반영

- 총합 보존은 해석 가능성을 위한 제약으로 유지하되, 최우선 기준은 예측 성능이다.
- 이후 비교/튜닝에서 `count calibration`은 참고 축으로 두고, 최종 선택 기준은 탐지/랭킹 성능을 우선한다.

## merge 해석

- 현재 full-scene merge의 lifetime 가중평균 자체는 물리적으로 말이 된다.
- 이유는 겹치는 cluster들이 서로 다른 독립 선박 집합이라기보다, 같은 장면 구조를 DRUID가 다른 lifetime으로 중복 포착한 것이기 때문이다.
- 따라서 겹치는 픽셀에 대해 각 cluster 출력을 그대로 더하면 같은 선박 근처 질량을 중복 집계할 가능성이 크다.
- 이 관점에서는 가중평균이 더 자연스럽고, `scene raster 적분값까지 척수 보존`을 강제하는 것은 현재 목적에선 과도할 수 있다.
