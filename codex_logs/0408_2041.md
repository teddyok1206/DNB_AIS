## 범위
- `PoissonNLL + Softplus` 설정의 이론적 합리성 정리
- 현재 Poisson 설정에서 `positive_weight=0, 10, 20, 30` 비교
- 해석 혼선을 막기 위해 `positive_weight=0, count_weight_alpha=0`인 완전 무가중치 기준도 추가 확인

## 이론적 판단
- 성능을 잠시 무시하고 target 의미만 보면, 현재 문제의 `y`는 픽셀별 선박 count이므로 `PoissonNLLLoss(log_input=False)`가 `MSE`보다 더 자연스럽다.
- 이유는 모델 출력이 비음수 intensity `lambda`가 되고, `y`가 count라는 해석과 직접 맞물리기 때문이다.
- 따라서 이론적 합리성만 놓고 보면 `Softplus + PoissonNLL` 쪽이 더 타당하다.
- 다만 실제 성능은 loss 의미와 별개로 데이터 희소성, weighting, epoch, sampling에 크게 좌우된다.

## 무가중치 기준
- 별도 확인 결과 `positive_weight=0, count_weight_alpha=0`이면 성능이 더 크게 무너졌다.
- 이 경우 주요 값은 다음과 같다.
  - `graph_topk_hit_rate=0.0127`
  - `scene_topk_hit_rate=0.0150`
  - `heatmap_max=0.0169`
  - `pred_sum_ratio=0.9355`
- 즉 Poisson에서도 양성 쪽을 어느 정도 밀어주는 weighting이 사실상 필요하다.

## 해석
- 현재 Poisson 설정 안에서는 `positive_weight=0`보다 `10~30`이 확실히 낫다.
- `10`은 scene top-k는 가장 높지만 graph top-k가 낮다.
- `20`은 graph top-k가 가장 높고 peak도 커서 이번 범위에선 가장 균형적이다.
- `30`은 peak와 총량은 더 커지지만 scene hit는 다시 떨어지고 mass 확산이 심해진다.
- 따라서 Poisson 계열만 놓고 보면 이번 실험 범위의 임시 추천값은 `positive_weight=20`이다.

## 현재 결론
- 이론적으로는 `PoissonNLL + Softplus`가 더 합리적이다.
- 하지만 현재 batch_demo 실험만 보면, 아직 이전 `MSE + ReLU + count_weight_yx6`를 성능으로 대체하진 못했다.
- Poisson을 계속 밀고 갈 거라면 다음 튜닝 우선순위는 `positive_weight=20` 근처 재확인, `count_weight_alpha` 재스윕, epoch 증가다.
