## sum-preserving edge-decay GT

- `y_edge_decay`를 source별 정규화 방식으로 변경했다.
- 각 양성 source node에 대해 hop kernel `(1.0, 0.6, 0.2)`를 먼저 만들고, 그 kernel 총합으로 나누어 source count가 퍼진 뒤에도 총합이 보존되도록 구현했다.
- 따라서 여러 source가 겹치더라도 source별 기여 총합은 각각 원래 척수와 같고, 전체 graph/scene 합도 원래 척수 합과 다시 연결된다.

## 검증 결과

- 검증 파일: [sum_preserving_validation.json](/Users/jungtaeuk/Desktop/SATGEO/[DNB_AIS]/outputs/DNB_GAT_v1/A2025001_1754_021/batch_demo/sum_preserving_validation.json)
- `graph_count=125`
- `raw_scene_sum=160.0`
- `raw_graph_sum=80.0`
- `edge_graph_sum=79.99999988079071`
- `max_abs_graph_diff=2.384185791015625e-07`

해석:

- DRUID patch graph 내부에서 supervision으로 쓰는 `y_edge_decay` 총합이 사실상 raw point GT 총합과 일치한다.
- 따라서 이제 `y_edge_decay`는 더 이상 단순 영향도 맵이 아니라, 합이 보존된 `척/노드` 분포로 해석할 수 있다.
- scene 전체 raw sum `160`과 graph raw sum `80`이 다른 이유는 DRUID contour 밖의 GT source는 graph 세트에 포함되지 않기 때문이다. 이 차이는 sum-preserving 구현의 문제가 아니라 patch coverage 문제다.
