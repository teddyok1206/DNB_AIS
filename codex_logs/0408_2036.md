## 해석
- 이번 batch 기준으로는 새 `PoissonNLL + Softplus`가 이전 `MSE + count_weight_yx6`보다 명확히 낫지 않았다.
- `graph_topk_hit_rate`가 `0.0759 -> 0.0380`, `scene_topk_hit_rate`가 `0.0526 -> 0.0451`로 떨어졌다.
- `heatmap_max`도 `0.2404 -> 0.2015`로 낮아져 peak 형성은 오히려 약해졌다.
- `pred_sum_ratio`는 `15.67 -> 14.58`로 약간 줄었지만, diffuse prediction 문제가 해소됐다고 보기는 어렵다.
- `train_loss` 숫자 자체는 MSE와 PoissonNLL 사이에서 직접 비교할 수 없다. 비교 의미가 있는 것은 top-k hit, peak, sum ratio 같은 출력 지표다.

## 현재 판단
- 이론적으로는 PoissonNLL이 count target과 더 잘 맞지만, 현재 batch_demo 세팅에서는 바로 성능 이득으로 이어지지 않았다.
- 원인은 짧은 epoch, 강한 희소성, 현재 positive weighting scale이 Poisson 쪽에 아직 맞지 않는 점일 가능성이 크다.
- 따라서 지금 코드 기본값은 요청대로 `Softplus + PoissonNLLLoss`로 바꿨지만, 성능 기준으로는 아직 기존 MSE 설정을 완전히 대체했다고 보긴 어렵다.

## 남은 이슈
- Poisson loss에서는 `positive_weight`, `count_weight_alpha`, `count_sum_lambda`, epoch 수를 다시 조정할 필요가 있다.
- 현재 비교는 batch_demo 1장 기준이므로, 데이터가 더 많아질 때 우위가 바뀌는지는 아직 모른다.
