## weighting 두 종류 차이

현재 학습 가중은 [sub_module/dnb_gat_pipeline.py](/Users/jungtaeuk/Desktop/SATGEO/[DNB_AIS]/sub_module/dnb_gat_pipeline.py) 의 `train_gat()` 안에서 이렇게 적용된다.

```python
weights = torch.ones_like(target)
weights = weights + (target > 0).float() * positive_weight
weights = weights + target * count_weight_alpha
```

차이는 명확하다.

- `positive_weight`
  - `target > 0`인 모든 node에 같은 추가 가중을 준다.
  - 값 크기와 무관하게 “양성이면 일괄적으로 더 세게 본다”는 의미다.
  - 예를 들어 `positive_weight=20`이면
    - target `0` -> weight `1`
    - target `0.2` -> weight `21`
    - target `0.6` -> weight `21`
    - target `1.0` -> weight `21`

- `count_weight_alpha`
  - target 값이 클수록 비례해서 더 큰 가중을 준다.
  - 즉 “중심점처럼 강한 GT를 더 세게, 주변처럼 약한 GT는 덜 세게” 본다.
  - 예를 들어 `count_weight_alpha=20`이면
    - target `0` -> weight `1`
    - target `0.2` -> weight `5`
    - target `0.6` -> weight `13`
    - target `1.0` -> weight `21`

정리하면:

- `positive_weight` = 양성/음성 구분용
- `count_weight_alpha` = 양성 내부에서도 강도 차등 반영용

현재처럼 `y_edge_decay`를 쓰는 상황에서는 두 가중이 같은 역할이 아니다. `positive_weight`는 퍼진 모든 양성 node를 똑같이 밀고, `count_weight_alpha`는 중심부를 더 강하게 밀어준다.
