## Pipeline Design Implemented
- `SceneRaster`: loads GeoTIFF and preserves geocoding metadata.
- `GroundTruthResolver`: uses existing geojson if present and can regenerate it from `ships.db` plus `metadata_JPSS-2.csv` if missing.
- `DruidClusterStore`: runs DRUID, keeps `cluster_id` and `lifetime`, reconstructs irregular contour masks, and filters nested clusters.
- `GraphBuilder`: converts each contour patch into a PyG graph where each pixel node has `[brightness, local_x, local_y]`.
- `GATDensityRegressor`: PyG `GATv2Conv` based regression model with non-negative output.
- `SceneAssembler`: merges cluster predictions back onto the source grid with lifetime-weighted averaging and writes GeoTIFF.

## Important Implementation Choices
- Default notebook mode is `batch_demo` because it is fully validated and keeps runtime bounded.
- `kr_full_scene` mode is already wired into the notebook configuration and uses DRUID cutup mode plus a cluster cap.
- I did not force rectangular crops for the GNN path. The graph is built directly on irregular contour masks, which matches your stated preference.
- To avoid DRUID contour-generation failures, contour masks are reconstructed from DRUID `Birth/Death/seed/bbox` outputs rather than relying on DRUID's fragile helper path.
